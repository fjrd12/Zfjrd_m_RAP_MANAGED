# Zfjrd_m_RAP_MANAGED

Example of a **managed scenario** for **RAP ABAP**.

## Overview

This repository provides a minimal reference structure for a managed RAP business object:

- CDS root view entity
- Managed behavior definition
- Behavior projection
- Service definition and binding

## Example snippets

### 1) CDS root view entity
```abap
@AccessControl.authorizationCheck: #NOT_REQUIRED
@EndUserText.label: 'Travel'
define root view entity ZI_TRAVEL as select from ztravel
{
  key travel_id      as TravelId,
      agency_id      as AgencyId,
      customer_id    as CustomerId,
      begin_date     as BeginDate,
      end_date       as EndDate,
      booking_fee    as BookingFee,
      total_price    as TotalPrice,
      currency_code  as CurrencyCode,
      overall_status as OverallStatus
}
```

### 2) Managed behavior definition
```abap
managed implementation in class zbp_i_travel unique;

define behavior for ZI_TRAVEL alias Travel
persistent table ztravel
lock master
authorization master ( instance )
{
  create;
  update;
  delete;

  field ( readonly ) TravelId;
}
```

### 3) Projection behavior
```abap
projection;

define behavior for ZC_TRAVEL alias Travel
{
  use create;
  use update;
  use delete;
}
```

### 4) Service definition
```abap
@EndUserText.label: 'Travel Service'
define service ZUI_TRAVEL {
  expose ZC_TRAVEL as Travel;
}
```

## Notes

- In a managed RAP scenario, standard CRUD handling is generated by the RAP framework.
- Add validations, determinations, and actions in behavior implementation as needed for business logic.
